# Inverse PINN

This directory contains the implementations of the inverse Physics-Informed Neural Networks (PINNs) used in the numerical experiments presented in our paper. The inverse PINNs reconstruct the unknown permeability field from synthetic velocity and pressure observations generated by the corresponding forward PINN models.

## Directory Structure

The implementations are organized according to the three permeability configurations considered in this work:

* 1K – Homogeneous permeability case.
* 2K – Single permeability anomaly case.
* 3K – Two permeability anomaly case.

Each subdirectory contains the implementations for the different inverse PINN variants investigated in the paper, including different regularization strategies.

## Input

Each inverse PINN implementation requires the corresponding forward PINN output file generated for the same permeability configuration. This file contains:

* Computational coordinates ((x,y))
* Synthetic velocity observations ((u_x,u_y))
* Synthetic pressure observations ((p))

These data are used to reconstruct the unknown permeability field while simultaneously enforcing the governing Darcy equations and boundary conditions. Additional input files, if required for a specific inverse PINN variant, are described in the corresponding subdirectory.

## Output

Each implementation reconstructs the permeability field and generates:

* A trained inverse PINN model (`.pth` file).
* Predicted permeability values.
* Predicted velocity and pressure fields.
* Output files used for visualization and quantitative error analysis.

## Requirements

The implementations require:

* Python 3.10 or later
* PyTorch
* NumPy
* Pandas
