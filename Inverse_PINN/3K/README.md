#3K: Two Permeability Anomalies

This directory contains the inverse PINN implementations for reconstructing the permeability field for the two-permeability-anomaly case. The following regularization strategies are investigated:

* InversePINN_3K_noreg.py – No regularization.
* InversePINN_3K_L1.py – L1 regularization.
* InversePINN_3K_L2.py – L2 regularization.
* InversePINN_3K_TV.py – TV regularization.
* InversePINN_3K_L1TV.py – L1+TV regularization.
* InversePINN_3K_L2TV.py – L2+TV regularization.

The network architecture, training procedure, and numerical setup are identical across all six implementations. The only difference is the regularization strategy used during training.

Input

Each regularization strategy uses one forward PINN output file as input, for example:

forward_output_3K_run1.csv

This file contains:

* Computational coordinates (x, y)
* Synthetic velocity observations (u_x, u_y)
* Synthetic pressure observations (p)

The synthetic observations are generated by the corresponding forward PINN and are used to reconstruct the unknown permeability fields.

Runs

For each regularization strategy, we perform 10 independent inverse PINN runs using random seeds 1–10. These repeated runs are used to evaluate the robustness and statistical consistency of the inverse reconstruction.

Output

Each implementation generates:

A trained inverse PINN model (`.pth` file).
A CSV file containing the reconstructed permeability field.
Predicted velocity and pressure fields.
Output files used for visualization and quantitative error analysis.
